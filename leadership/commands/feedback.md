Use the `leadership-people-leader` skill to coach the user through delivering a specific piece of feedback.

## When the user invokes this

Ask one question if not provided: **"Who is the feedback for, and what's the situation?"**

## Steps

1. **Get the situation, the behavior, the impact** — the SBI raw material:
   - What specifically did they do (or not do)? When? In what context?
   - What was the impact — on the team, the client, the project, the person themselves?
   - What's the pattern, if any (one-off vs. recurring)?
2. **Sort:** is this corrective, positive, or developmental? Is it one-off (immediate, brief) or substantive (warrants a slot in the next 1:1)?
3. **Calibrate the quadrant** (Radical Candor):
   - Care personally — what's the underlying investment in this person's growth?
   - Challenge directly — what's the actual message, unsoftened?
   - Test for ruinous empathy (over-padding) and obnoxious aggression (caring missing)
4. **Draft the actual words.** SBI structure:
   - "When [specific behavior in specific context]..."
   - "...the impact was [specific consequence]..."
   - For corrective: "Could you [specific behavior next time]?" + listen
   - For positive: "Thank you / keep doing that" + name the pattern
5. **Anticipate the response.** What's their likely defense, and how does the user want to handle it?
6. **Decide the channel and timing.** Real-time micro-feedback in a hallway? Specific 1:1 slot? Scheduled conversation?
7. **Don't deliver the feedback for the user.** This command produces the script and the framing; the user delivers it.

## Tone

- Direct. No padding. Specific.
- Behavior-focused, not character-focused.
- Future-focused for corrective ("could you do this next time"), not blame-focused
- For sustained corrective patterns, surface the at-stake explicitly without softening

## Don't

- Default to the open-faced shit sandwich (Nightingales explicitly rule this out)
- Confuse "kind" with "vague" — vague feedback is unkind because it doesn't help
- Drift into general feedback theory — produce *this specific feedback*

## Pairs with

- `/1on1` — to log the conversation after it happens
- `/decision` — if the feedback leads to a decision (improvement plan, scope change, exit)
- `/perspective` — if the user is reactive about the feedback they have to give and needs equanimity first
