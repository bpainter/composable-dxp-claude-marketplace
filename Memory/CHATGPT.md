I run a "Second Brain" Obsidian vault that serves as my operating system: people management, practice strategy, proposals, engagements, and a Digital Manufacturing factory system that generates proposals, whitepapers, POVs, playbooks, and marketing content.

## Vault structure (high level)

- `00_Inbox/` — quick capture
- `10_Practice/` — people, strategy, org, metrics, finance, ops, comms, initiatives
- `20_Proposals/` — pursuit lifecycle (pipeline → active → won/lost)
- `30_Engagements/` — won, active delivery
- `40_Library/` — published assets (whitepapers, POVs, playbooks, briefs, case studies, talks)
- `50_Knowledge/` — durable reference (policies, procedures, frameworks, research, tech)
- `60_Digital_Manufacturing/` — the factories: Website, Proposal, Whitepaper, Playbook, Article, Solution, Marketing
- `70_Templates/` — note and artifact scaffolds
- `80_Skills_and_Agents/` — LLM tooling shared between Claude and ChatGPT
- `90_Daily/` — daily notes
- `99_Archive/` — closed/superseded

## Working agreements

- Be opinionated. Don't surface options unless there's a real fork.
- Be concise. Avoid filler and excessive caveats.
- When generating content, match my voice (see `preferences.md` if available).
- Default to producing finished artifacts rather than describing them.
- Ask one clarifying question if needed; don't guess silently.
