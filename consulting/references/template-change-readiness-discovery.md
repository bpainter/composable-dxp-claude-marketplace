# Template — Change Readiness & Discovery

**Sources:** Robbins & Judge, *Essentials of Organizational Behavior* (Pearson, 2018) — see [`book-essentials-of-organizational-behavior.md`](book-essentials-of-organizational-behavior.md). Block, *Flawless Consulting* (Pfeiffer, 2011) — see [`book-flawless-consulting.md`](book-flawless-consulting.md). Plus Lewin (unfreeze-change-refreeze) and Kotter (8 steps) as classical references.

A change-readiness assessment + discovery worksheet that combines OB-grounded readiness diagnostics (Robbins/Judge) with Block's discovery-interview structure for the people side. Use it before launching any change initiative — digital transformation, restructuring, M&A integration, new ways of working.

## When to use it

- Before scoping a change-management workstream inside a consulting engagement
- When a transformation has stalled and resistance is the suspected cause
- Before any rollout that affects how 50+ people do their work
- During M&A planning (Day 1 readiness, post-Day-100 cultural integration)

## Section 1: The Three Frameworks This Template Sits On

Pick the frame that fits your engagement. Most engagements use Lewin as the macro arc with Kotter as the operating model and ADKAR as the individual-level diagnostic.

### Lewin's three phases (the macro arc)

1. **Unfreeze** — disturb the current equilibrium, increase driving forces and reduce restraining forces (force-field analysis)
2. **Change/Move** — implement the new state, including learning new behaviors
3. **Refreeze** — institutionalize the new state, reinforce, embed in systems

### Kotter's 8 steps (the operating playbook, mapped to Lewin)

| # | Step | Lewin phase |
|---|---|---|
| 1 | Establish a sense of urgency | Unfreeze |
| 2 | Form a guiding coalition | Unfreeze |
| 3 | Develop a vision and strategy | Unfreeze |
| 4 | Communicate the change vision | Unfreeze |
| 5 | Empower broad-based action | Move |
| 6 | Generate short-term wins | Move |
| 7 | Consolidate gains, produce more change | Move |
| 8 | Anchor new approaches in culture | Refreeze |

### ADKAR (the individual-level diagnostic)

For each person affected: **A**wareness of the need → **D**esire to participate → **K**nowledge of how to change → **A**bility to implement → **R**einforcement to sustain. Diagnose where individuals are stuck — most resistance traces to one missing element.

## Section 2: Change Readiness Assessment

Before anything else, score the organization's readiness on five dimensions. Each scored 1–5; total ≥18 indicates strong readiness, ≤12 indicates significant work needed before launching.

| Dimension | What "1" looks like | What "5" looks like | Score |
|---|---|---|---|
| **Sponsor commitment** | No senior sponsor; nobody owns the change visibly | CEO-level sponsor actively leading, present at milestones | / 5 |
| **Burning platform / urgency** | "Things are mostly fine"; status quo is comfortable | Clear external or internal trigger; cost of inaction is widely understood | / 5 |
| **Past change track record** | Prior changes failed or stalled; "change fatigue" is real | Recent changes succeeded; the org has change-delivery muscle | / 5 |
| **Coalition strength** | One champion; siloed support | Cross-functional coalition with credible voices in every affected area | / 5 |
| **Capacity to absorb** | Multiple changes already in flight; teams stretched | Adequate resourcing; this change has dedicated bandwidth | / 5 |
| **Total** | | | / 25 |

If total ≤ 12 → before any rollout, invest in raising the score. Often the right move is a smaller, contained pilot to build evidence before broader change.

## Section 3: Force-Field Analysis (Lewin's pre-Unfreeze diagnostic)

For the specific change in question, list the forces pushing for the change and pushing against it. The ratio matters more than the totals — equal forces produce stasis even if both are large.

```markdown
# Force-Field — [Specific Change]

## Driving forces (push for the change)
| Force | Strength (1-5) | Notes |
|---|---|---|
| External market pressure | | |
| Competitor moves | | |
| Customer expectations | | |
| Senior leader commitment | | |
| Cost of status quo | | |
| | | |

**Total driving:** / 30

## Restraining forces (push against the change)
| Force | Strength (1-5) | Notes |
|---|---|---|
| Past change failures | | |
| Skill gaps | | |
| Cultural attachment to current state | | |
| Fear of role/job change | | |
| Resource constraints | | |
| Active blockers | | |
| | | |

**Total restraining:** / 30

## Net:
Driving - Restraining = 

## Strategy
- Top 2 driving forces to amplify: ___
- Top 2 restraining forces to reduce: ___
```

Lewin's insight: reducing restraining forces is usually more effective than amplifying driving forces. Adding more "burning platform" rhetoric to a team already on edge backfires.

## Section 4: Stakeholder Resistance Map

Robbins & Judge identify common sources of resistance (Ch. 17, pp. 315–323):

- **Habit** — "this is how we've always done it"
- **Security** — fear of job/role change
- **Economic factors** — direct compensation impact
- **Fear of the unknown** — uncertainty about future state
- **Selective information processing** — hearing what fits prior beliefs
- **Structural inertia** — systems, rules, processes hold stable
- **Group inertia** — peer norms reinforce status quo
- **Threat to expertise** — change devalues current skills
- **Threat to power** — change shifts authority

Block adds the consultant's diagnostic vocabulary (13 forms of resistance, including the flood of detail, attack, compliance, methodology, "we've tried that", etc., Ch. 12).

```markdown
# Resistance Map — [Specific Change]

| Stakeholder/Group | Resistance source(s) | How it manifests | Engagement strategy |
|---|---|---|---|
| | | | |
| | | | |
| | | | |
```

For each, the strategy is usually one of: educate (Awareness), motivate (Desire), train (Knowledge), enable (Ability), or reinforce (Reinforcement) — i.e., apply ADKAR.

## Section 5: Discovery Interview Guide for Change

Block's 5-layer discovery (`template-engagement-scoping.md`) adapted for change work. Conduct 8–12 interviews across affected functions and levels.

```markdown
# Change-Readiness Discovery Interviews

## Selection criteria (8-12 interviews typical)
- 1–2 from sponsor/exec layer
- 2–3 from middle management (often where change dies)
- 3–4 from frontline (who actually has to do the change)
- 1–2 from adjacent functions (whose work depends on this)
- 1 known skeptic
- 1 known champion (less critical to interview but useful to understand the "yes" framing)

## Interview structure (45-60 min each)

**Section 1: Their understanding of the change (10 min)**
- "What's your understanding of what's being proposed?"
- "When did you first hear about this?"
- "From whom?"
- (This surfaces communication-cascade gaps and ADKAR Awareness.)

**Section 2: Their take on whether it's needed (10 min)**
- "What problem do you think this is trying to solve?"
- "On a scale of 1-10, how urgent is this?"
- "If we did nothing, what would happen?"
- (ADKAR Desire — do they actually want this?)

**Section 3: How it affects them personally (15 min)**
- "What changes for you, day-to-day?"
- "What are you most worried about?"
- "What are you most excited about?"
- "Who in your network is most affected?"
- (Block's personal layer — the political and emotional reality.)

**Section 4: What's been tried before (10 min)**
- "Has the org tried something like this before?"
- "What happened?"
- "What would you do differently this time?"
- (Past change track record; structural inertia signals.)

**Section 5: Their wishes for how it should run (10 min)**
- "If you were running this change, what would you do first?"
- "What would convince you it's working?"
- "What would tell you it's failing — and what should we do then?"

**Closing**: "What didn't I ask that I should have?" Confirm next steps, thanks.

## Synthesis
After all interviews:
- Where on Lewin's arc is each affected group? (most are stuck at Unfreeze if change isn't going well)
- What's the dominant resistance source per group?
- Where are the unexpected pockets of support / opposition?
- What's the gap between sponsor's understanding and frontline's understanding?
```

## Section 6: Quick-Win Plan (Kotter Step 6)

Quick wins are the highest-leverage tactic in change. Plan them deliberately — don't hope they emerge.

```markdown
# Quick Wins — [Change Name]

| Win | What it demonstrates | Audience | When | Owner |
|---|---|---|---|---|
| | | | | |
| | | | | |
```

Criteria for a good quick win (Kotter):
- Visible and unambiguous
- Clearly tied to the change
- Achievable within 6–18 months
- Resourced

## Pitfalls

- **Skipping the readiness assessment.** Launching change without scoring readiness is the #1 cause of failed transformations. Robbins/Judge: most failures trace to a Lewin step that was skipped — usually Unfreeze.
- **Treating change as a communications problem.** Communication is necessary but insufficient. ADKAR is sequential; you can't communicate-your-way past missing Knowledge or Ability.
- **The "town hall and we're done" trap.** Kotter Step 4 (communicate the vision) is one step among 8. Many leaders treat it as the whole change program.
- **Ignoring middle managers.** Robbins/Judge note middle managers are the most consistent source of change failure because they're squeezed between the vision and the daily work. Engage them as a coalition (Kotter Step 2), not as an audience.
- **Past-failure amnesia.** Most orgs have failed at change before. Surface those memories explicitly; they predict resistance better than any survey.
- **Treating resistance as opposition.** Block's framing: resistance is data. Skeptics are giving you the test the change must pass. Don't crush; convert.
- **Quick-win theater.** Manufactured wins that aren't actually tied to the change erode credibility. Real wins, even small, are better than impressive vapor.
- **Stopping at "rollout."** Kotter Step 8 (anchor in culture) and Lewin's Refreeze are the longest, most-skipped phase. Most "successful changes" that backslide quietly stopped at Move.
