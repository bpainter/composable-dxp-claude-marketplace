# Book digest: *Gamestorming*

**Authors**: Dave Gray (founder of XPLANE), Sunni Brown (founder of Sunni Brown Ink and the Doodle Revolution), James Macanufo. Published by O'Reilly Media in 2010. A v2.0 update by Gray and Brown was published in 2024 with ~95 games (vs. the original 80+).
**Companion site**: [gamestorming.com](https://gamestorming.com/) — the open catalog with community-contributed games beyond the book.

**What this book is**: A practitioner's playbook of named games for collaborative work. Each game is a structured activity with a defined object of play, time bound, rules, and an artifact the group produces. The book argues that *play* is a serious tool for innovation, problem-solving, and team alignment — not a frivolous addition to "real" work.

This is the canonical anchor for the `facilitation-gamestorming-coach` skill in this plugin. Per-game digests for the full ~110-game catalog (book + site) live in [`gamestorming-catalog.md`](../skills/facilitation-gamestorming-coach/references/gamestorming-catalog.md).

## The headline argument

Knowledge work is fundamentally different from industrial work. The challenges — sense-making, designing, deciding under uncertainty, aligning across functions — don't yield to the linear, predict-then-execute methods that dominate management literature. They yield to **structured play**: bounded activities with rules and artifacts that let groups think collectively in ways that ordinary meetings prevent.

A game in the Gamestorming sense is not a metaphor. It's a designed activity with the same elements that make any game work — a clear *object of play* (what winning looks like), *players* (who's in), *rules* (constraints that produce the desired thinking), *time bound* (when the game ends), and an *artifact* (what the play produces).

## Structure of the book

### Part 1 — Theory (Chapters 1–3)

The intellectual scaffolding. Why games work, what makes a good game, how to design and run them.

- **Chapter 1: Opening up: the world of Gamestorming** — the case for play as a serious tool; the difference between knowledge work and industrial work
- **Chapter 2: Game design** — the seven essentials of a good game (object of play, game world, participants, rules, opening, exploring, closing)
- **Chapter 3: Core games of Gamestorming** — the ten games to learn first

### Part 2 — The catalog (Chapters 4–8)

The games organized by purpose:

- **Chapter 4: Games for opening** — generative, divergent, set-context games
- **Chapter 5: Games for exploring** — sort, develop, test, deepen
- **Chapter 6: Games for closing** — converge, decide, commit
- **Chapter 7: Games for designers** — design-specific games
- **Chapter 8: Games for fresh thinking** — break out of stuck patterns

### Closing

Tips on running games, building your own, organizational adoption.

## The seven essentials of a good game

From Chapter 2 — the design checklist for every game:

1. **Opening** — how the game starts; what frame the players need
2. **Object of play** — what the game produces; the "win condition"
3. **Game world** — the physical or virtual space the game inhabits (board, canvas, room)
4. **Players** — who participates; recommended size
5. **Rules** — the constraints that produce the desired thinking
6. **Closing** — how the game ends; how the artifact gets captured
7. **Time bound** — duration that fits the cognitive load and stakes

When designing a new game (or modifying an existing one), every essential must be answered.

## The Core Games

The ten games the book identifies as the foundation:

| Game | Purpose |
|---|---|
| **7Ps Framework** | Designing meetings (Purpose, Product, Participants, Probable issues, Process, Preparation, Practical concerns) |
| **Affinity Map** | Clustering raw ideas into themes |
| **Bodystorming** | Physical-movement ideation |
| **Card Sort** | Categorizing concepts |
| **Dot Voting** | Convergent prioritization |
| **Empathy Map** | Externalizing a user's perspective |
| **Forced Ranking** | Strict prioritization with no ties |
| **Post-Up** | Silent generation on sticky notes |
| **Storyboard** | Narrative visualization |
| **WhoDo** | Surfacing actors and actions in a system |

Internalize these first. They appear in nearly every Gamestorming-driven session.

## The three-movement session

Gamestorming's framing for sequencing games:

- **Opening** — divergence, possibility, surfacing what's known
- **Exploring** — examining, sorting, deepening, prioritizing
- **Closing** — converging, deciding, committing

This maps directly onto Scan/Focus/Act (Collaboration Code) and divergence/insight/convergence (general facilitation). Same shape, different vocabulary.

## How Gamestorming complements the rest of this plugin

| Source | Strength | When to reach for it |
|---|---|---|
| **Gamestorming** | Crisp named games, web-shareable, easy to teach to client teams | Most ideation, prioritization, alignment moments in workshops |
| **Collaboration Code Tools** | Wavespace-tradition modules with deeper variations and SDT context | Multi-day sessions where you need module-by-module documentation |
| **Strategyzer canvases** | Strategy and innovation canvases as facilitation surfaces | When the central artifact is a canvas (BMC, Value Prop, etc.) |
| **Design Thinking Playbook moves** | DT-specific facilitator moves | Discovery sprints, empathy work, prototype-and-test cycles |

Most named games appear in at least two of these traditions. Empathy Map is in Gamestorming, the Collaboration Code Tools, and Design a Better Business. Pick the source that matches the room's vocabulary.

## How to reach for this in real work

- **Designing a session** — open the catalog by movement (opening / exploring / closing) and pick games whose object of play matches what you need at each phase.
- **Stuck mid-session** — open the "Games for fresh thinking" or "Games for problem-solving" categories on gamestorming.com and pick a 15-minute reset.
- **Coaching a colleague** — start them on the ten Core Games; expand from there as their facilitation muscle grows.
- **Teaching a client team** — Gamestorming language is more accessible than wavespace vocabulary; the "game" framing lowers resistance to participatory work.
