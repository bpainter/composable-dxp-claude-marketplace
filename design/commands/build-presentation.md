Start a 16:9 presentation deck (PowerPoint, Keynote, Gamma, Google Slides) with system thinking before any slide goes on the canvas.

Steps:

1. **Calibrate via [[design-taste]]** — dials, intent-first protocol, four required outputs, register selection.

2. **Define the deck system via [[design-presentation]].** Write `deck-system.md` in the WIP run folder before any slide content. Cover all seven elements: master grid, type scale, color tokens, slide archetypes, density discipline, transitions, speaker-notes discipline. Reference [[surface-deck-16x9]] for full format specs.

3. **Build the masters in `pptx`.** One master slide per archetype (Cover, Section divider, Big claim, Headline+visual, Comparison, Process, Quote, Data viz, Architecture, Q&A, Closing commitment).

4. **Plan the slide order.** Outline view: slide titles alone should tell the story.

5. **Compose slides** using the masters. For supporting work:
   - Imagery → [[design-imagery]].
   - Diagrams → [[design-visualization]].
   - Charts → [[shadcn-chart-inventory]].
   - Icons → [[design-iconography]].

6. **Speaker notes pass.** Every non-self-explanatory slide gets notes.

7. **Run the four checks** ([[design-taste]] — swap, squint, signature, token).

8. **Run [[pre-delivery-checklist]]** plus deck-specific gates ([[surface-deck-16x9]] checklist):
   - Cover earns the next 10 minutes.
   - Closing slide says something specific (not "Thank you" or "Questions?").
   - No five-pillar methodology slide.
   - No headshot grid.
   - Body type ≥ 18pt.

9. **Hand off** the file plus speaker-notes script.

Inputs to confirm before starting:
- Audience (specific person, not "executives").
- Intent (what they should do/decide after).
- Length target.
- Stylistic register (default for Slalom: premium-restrained for executive, swiss-modern for technical).
- Brand inputs (logo, color tokens, type system).
