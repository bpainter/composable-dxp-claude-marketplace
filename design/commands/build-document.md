Start an 8.5×11 long-form document with editorial system thinking before any content layout.

Steps:

1. **Calibrate via [[design-taste]]** — dials, intent-first, register (default editorial-tech for technical thought leadership; premium-restrained for executive briefings).

2. **Define the document system via [[design-document]].** Write `document-system.md` in the WIP folder before layout. Cover format/grid, type scale (10–12pt body for 8.5×11), color system, section archetypes, header/footer system, citation style. Reference [[surface-document-letter]] for full specs.

3. **If manuscript exists, read end-to-end.** Note where pull quotes, callouts, and figures naturally fit. If it doesn't exist, route to the writing skill upstream.

4. **Build the document master** in docx, InDesign, or PDF scaffold. One template covering all archetypes (Cover, Exec summary, TOC, Section open, Body prose, Pull quote, Callout, Figure, Citations, Back cover).

5. **Lay out the content** using masters. Supporting work:
   - Cover and figure imagery → [[design-imagery]].
   - Diagrams → [[design-visualization]].
   - Charts → [[shadcn-chart-inventory]] (export to PNG/SVG for embed).
   - Callout and section icons → [[design-iconography]].

6. **Run the four checks** ([[design-taste]]).

7. **Run [[pre-delivery-checklist]]** plus document-specific gates ([[surface-document-letter]] checklist):
   - Cover earns the document (no full-bleed Unsplash defaults).
   - Exec summary makes a claim, doesn't list contents.
   - No "our approach" five-pillar page.
   - Body type ≥ 10pt for print, ≥ 11pt for screen.
   - Citation style consistent throughout.
   - All hyperlinks resolve.
   - Accessible PDF tags if exporting.

8. **Hand off to `docx` or `pdf` skill** for file generation.

Length conventions (Slalom Composable DXP defaults):
- Whitepaper: 8–15 pages.
- POV: 3–5 pages.
- Solution brief: 4–8 pages.
- Playbook: 12–30 pages.
- Case study: 1–4 pages.
